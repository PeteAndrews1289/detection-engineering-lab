# Security policy

## Reporting

Please report a suspected vulnerability through GitHub private vulnerability reporting when available. Do not open a public issue containing credentials, personal data, raw enterprise telemetry, or exploit payloads.

## Data policy

This repository accepts only synthetic or deliberately sanitized fixtures. Contributions must not include:

- access keys, tokens, passwords, or session material;
- real account, tenant, customer, employee, or host identifiers;
- raw production logs;
- live malicious payloads or operational infrastructure; or
- evidence that cannot be redistributed safely.

The automated checks are guardrails and do not replace human review.
